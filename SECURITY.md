# Security

This project is intended for read-only local inspection of Preply accounts you own or are authorized to inspect.

## What Not To Commit

- Preply exports from `data/`
- Browser cookies or session headers
- Screenshots containing student names, messages, or receipts
- API tokens, passwords, or 1Password item references

## Reporting

If you find a credential exposure or a behavior that mutates Preply account state, open a private security report or contact the maintainer before publishing details.
